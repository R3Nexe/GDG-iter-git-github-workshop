##  What You'll Practice

By completing this activity, you will have done a **real-world Git workflow** end to end:

```
Fork → Clone → Edit → Commit → Push → Pull Request
```

This is exactly how developers contribute to open source projects every day.

##  Quick Steps (full detail in [`CONTRIBUTING.md`](./CONTRIBUTING.md))

```bash
# 1. Fork this repo on GitHub (click the Fork button top-right)

# 2. Clone your fork
git clone https://github.com/<your-username>/GDG-iter-git-workshop.git
cd GDG-iter-git-workshop

# 3. Create a branch with your name
git checkout -b attendance/<your-name>

# 4. Add your row to ATTENDEES.md

# 5. Stage and commit
git add ATTENDEES.md
git commit -m "feat: add <your-name> to attendees"

# 6. Push to your fork
git push origin attendance/<your-name>

# 7. Open a Pull Request on GitHub
```
